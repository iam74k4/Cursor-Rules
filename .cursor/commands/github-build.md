# github-build

GitHub Actionsを使用した自動ビルド・リリースのワークフローファイルを生成します。

## 使用方法

`/github-build` コマンドを実行すると、プロジェクトの言語や構造に応じて適切なワークフローファイルを生成します。

## 生成されるワークフロー

### 基本機能

-   タグプッシュ時に自動ビルド・リリース
-   ビルド成果物（exe）をGitHub Releasesにアップロード
-   複数OS対応（Windows、macOS、Linux）
-   バージョン番号の自動管理（タグから自動抽出）

### 対応言語

-   Go
-   Rust
-   C# (.NET)
-   Python
-   Node.js

## 生成手順

1.  プロジェクトの言語を指定
2.  ビルドコマンドと出力ファイル名を確認
3.  ワークフローファイル（`.github/workflows/build.yml`）を生成
4.  必要に応じてカスタマイズ

## リリース方法

1.  タグを作成してプッシュ（セマンティックバージョニングに従う）:
    ```bash
    git tag v1.0.0
    git push origin v1.0.0
    ```

2.  GitHub Actionsが自動実行され、以下が実行されます:
    -   タグからバージョン番号を自動抽出（`v1.0.0` → `1.0.0`）
    -   バージョン情報をバイナリに埋め込み
    -   ビルド後にReleasesにexeがアップロード
    -   リリース名にバージョン番号を使用

## バージョン管理

### セマンティックバージョニング

-   `MAJOR.MINOR.PATCH` 形式（例: `1.2.3`）
-   `MAJOR`: 互換性のない変更
-   `MINOR`: 後方互換性のある新機能
-   `PATCH`: 後方互換性のあるバグ修正

### バージョンの自動抽出

ワークフローでは、タグ名（`v1.0.0`）から自動的にバージョン番号（`1.0.0`）を抽出し、ビルド時に使用します。

### 言語別のバージョン埋め込み

-   **Go**: `-ldflags "-X main.Version=$VERSION"`
-   **Rust**: 環境変数またはビルド設定
-   **C#**: ビルド引数でバージョン指定
-   **Node.js**: `package.json` の `version` を更新

## カスタマイズ

生成されたワークフローファイルは、プロジェクトの要件に応じてカスタマイズ可能です：

-   ビルドコマンドの変更
-   成果物の名前変更
-   追加のビルドステップ
-   テストの実行
